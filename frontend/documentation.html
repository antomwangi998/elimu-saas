🏫 System Overview

ElimuSaaS is a comprehensive, cloud-based school management platform built specifically for Kenyan schools. It supports both 8-4-4 and CBC curricula and handles everything from student admissions to fee collection, exams, payroll and parent communication.

Fee Management
M-Pesa STK, Paybill, bank. Invoices, statements, clearance

EE/ME/AE/BE levels, learning areas, learner portfolios
✅
Attendance
Daily roll call, SMS alerts, monthly reports
📅
Timetable
Auto-generate conflict-free class schedules
💼

Staff payroll, PAYE, NHIF, NSSF, payslips
💬

Bulk SMS, WhatsApp, newsletters to parents

Architecture: Node.js/Express backend · PostgreSQL database · Vanilla JS SPA frontend · Kotlin Android app · Hosted on Render.com

🚀 First Login & Setup

1

Navigate to the login page

Go to the website and click Login. Enter your email, password, and school code (provided by your system administrator).

2

Change your temporary password

If this is your first login, you'll be prompted to set a new password. Use at least 8 characters with a mix of letters, numbers and symbols.

3

Complete your profile

Go to Settings → Profile to update your name, profile photo and notification preferences.

4

Explore your dashboard

Your dashboard is tailored to your role. Admins see school statistics; teachers see their classes; bursars see financial summaries.

🏫 School Setup

School Profile

Go to School Profile in the sidebar to configure your school's information:

  • Basic Info: School name, KNEC code, school type (primary/secondary/mixed), boarding type
  • Contact: Phone, email, website, physical address, county
  • Branding: Upload school logo and cover photo directly (no URL needed)
  • Principal Details: Principal's name and signature for report cards
  • Motto: Appears on report cards and broadsheets

Academic Setup

Under Academics → Classes & Subjects:

  • Create classes (Form 1–4 for secondary; Grades 1–8 for primary)
  • Add subjects and assign teachers to each class-subject combination
  • Set periods per week for timetable generation
  • Form 1: Up to 11 subjects. Forms 2–4: Up to 8 subjects

📝 Exams & Mark Entry

Creating an Exam Series

1

Go to Exams → + New Exam Series

Give it a name (e.g. "End of Term 1 2025"), select the type (opener, mid-term, end-term, mock), and choose which classes are included.

2

Teachers enter marks

Each teacher sees only their assigned subjects and classes. They enter marks per student, add remarks, and mark absent students.

3

Lock the series

Once all marks are entered, the principal or HOD locks the series. Locked series cannot be edited.

4

Generate broadsheets & report cards

Go to Broadsheet or Report Cards tab, select the series and class, and generate.

Grading Scale

Go to Settings → Grading to customise your grading scale. Presets available:

  • KCSE 8-4-4: A (75–100, 12pts) through E (0–24, 1pt) — 12-point scale
  • CBC: EE (Exceeding), ME (Meeting), AE (Approaching), BE (Below) — 4-level scale
  • Custom: Define any grading bands with your own labels and points

📋 Broadsheet

The broadsheet provides a Zeraki-style class-wide performance overview, printed in A3 landscape format.

  • Select exam series and class, then click Generate Broadsheet
  • Opens in a new window with school header (logo, name, address, motto)
  • Each student row shows marks and grade per subject, total, mean marks, mean points, mean grade, and class position
  • Subject performance analysis table at the bottom
  • Signature lines for Class Teacher, HOD/Dean, and Principal
  • Form 1 shows up to 11 subjects; Forms 2–4 show up to 8 subjects

💰 Fee Management

Setting Up Fee Structures

1

Create fee structure

Go to Fees → Structures → Create Structure. Name it (e.g. "Form 4 Boarding Term 1 2025") and add fee items: Tuition, Activity, Boarding, Exam, etc.

2

Assign to classes

Link the fee structure to specific classes and terms. Students in those classes automatically get the fee balance assigned.

3

Record payments

Click + Record Payment anywhere in the fees section. Search for the student, enter amount, select payment method.

Payment Methods Supported

  • M-Pesa STK Push: System sends a push to parent's phone — they enter PIN to pay
  • M-Pesa Paybill: Parent pays to school's Paybill number, enter receipt number
  • Bank Transfer: Enter bank reference number
  • Cash: Record cash payment, system generates receipt
  • Cheque: Record cheque payment with reference

📱 M-Pesa Integration

ElimuSaaS integrates with the Safaricom Daraja API for mobile money:

Paybill Setup (for Schools)

  1. Register a Safaricom Paybill number (Business Short Code)
  2. Provide the Paybill number and passkey to ElimuSaaS support
  3. Go to Settings → Integrations and enter your Daraja API credentials
  4. Test with a small payment before going live

M-Pesa STK Push Flow

  1. Bursar enters student, amount and parent's phone number
  2. System initiates STK Push via Daraja API
  3. Parent receives prompt on phone and enters M-Pesa PIN
  4. Payment confirmed automatically, receipt generated
Note: M-Pesa STK Push requires the parent's phone to be registered on M-Pesa and have sufficient balance. The school must have an active Safaricom Daraja API account.

✅ Attendance

Attendance is recorded daily per class. Go to Attendance in the sidebar:

  • Select class and date, then mark each student as Present, Absent, Late, Excused, or Sick
  • Bulk actions: mark all present, then uncheck absentees
  • SMS alerts automatically sent to parents of absent students (if SMS integration is configured)
  • Monthly attendance reports and percentage calculations per student
  • Attendance appears on report cards

📱 Android App

Download

Download the latest APK from the GitHub Releases page or from the home page of the website.

Installation

  1. Download the .apk file to your Android device
  2. Go to Settings → Security → Unknown Sources and enable it (or "Install unknown apps")
  3. Open the downloaded APK and tap Install
  4. Open ElimuSaaS and log in with your credentials

Offline Mode

The Android app caches recent data locally. When offline, you can still view:

  • Dashboard statistics (last loaded data)
  • Student list and basic info
  • Your timetable
  • Recent attendance records

Data entered offline will sync when connectivity is restored.

Connection Note: The server may take 30–60 seconds to wake up if it hasn't been used recently (Render free tier). The app will retry automatically. If login fails, wait 30 seconds and try again.

🔐 Security

  • HTTPS only: All traffic encrypted with TLS 1.3
  • JWT authentication: Short-lived access tokens (15 min) + refresh tokens
  • Bcrypt passwords: All passwords hashed with bcrypt (cost factor 10)
  • Tenant isolation: Every query scoped to school_id — School A cannot access School B's data
  • Rate limiting: 100 requests/15 min per IP on login endpoints
  • Android: Root detection, emulator detection, certificate pinning, ProGuard obfuscation
  • CSP headers: Content Security Policy prevents XSS attacks

Password Policy

  • Minimum 8 characters
  • Must contain letters and numbers
  • Must change temporary passwords on first login
  • Administrators can reset passwords for users below them in the hierarchy

⚙️ Super Admin Guide

The Super Admin role manages the entire ElimuSaaS platform across all schools.

Onboarding a New School

1

Create school

Go to All Schools → + New School. Enter school name, code, admin email, phone, county and type.

2

System creates admin account

An admin account is automatically created with a temporary password. Share the school code and credentials with the principal.

3

Activate subscription

Go to Subscriptions and activate a plan for the school. The school can also self-subscribe from their Settings page.

Suspending a School

If a school's subscription lapses, click Suspend on their profile. All users in that school will see a lockout page until the subscription is renewed.

Resetting Admin Passwords

Click on a school → Reset Admin Password. A temporary password is generated and shown to you — share it with the school admin securely.
